Core capabilities

Everything needed to generate, observe, and document TSN traffic behavior.

TSNPerf is now presented as a focused engineering product for lab validation, platform bring-up, and repeatable network performance checks.

MS

Multi-Stream Packet Generator

Transmit up to 8 VLAN and PCP traffic classes simultaneously using the --streams syntax, with one OS thread and one timing controller per stream.

  • Independent per-stream sequence numbering for loss detection
  • Separate TSN cycle gate and rate controller per class
  • Aggregated JSON output with full streams[] statistics
PG

Packet Generation and Loss Detection

Generate UDP traffic with configurable packet rate, packet size, and total packet count while embedding sequence numbers into every payload.

  • Gap-based packet loss detection without sender and receiver coordination
  • Same workflow for loopback testing and real Ethernet hardware runs
  • Consistent command-line operation for scripted validation
Q

Real 802.1Q Header Support

Use proper Ethernet-level VLAN tagging with EtherType 0x8100, TCI mapping for PCP, DEI, and VID, and inner EtherType 0x88B5 for TSNPerf traffic.

  • Configurable destination MAC for unicast or broadcast
  • Raw socket I/O through pnet_datalink
  • Switches can classify traffic by PCP directly from the frame header
TS

TSN Cycle Scheduler

Model simplified 802.1Qbv gate control behavior with cycle duration and open-slot timing defined in microseconds, milliseconds, or seconds.

  • Gate-controlled injection for slot-bounded transmission
  • Sleep plus spin-wait timing strategy for precision without full CPU saturation
  • Useful for testing scheduled traffic windows and cycle alignment
LM

Latency Measurement and Statistics

Measure one-way latency as receive timestamp minus send timestamp and compute min, max, average, jitter, and packet loss across all received traffic.

  • Jitter uses the RFC 3550 mean absolute deviation method
  • Loopback reveals OS scheduling and stack latency
  • PTP-synchronized hardware runs support true network latency analysis
WD

Live Dashboard and JSON Export

Launch a light-theme embedded dashboard from the receiver and monitor latency trends, histogram movement, per-stream breakdown, and live stat cards.

  • Single embedded HTML page with no separate frontend install
  • Adaptive histogram scaling and stream-aware charts
  • Structured JSON export for CI pipelines and regression tracking

Download and run

Get TSNPerf from Git and start testing.

Use Git as the delivery channel for the tool, then build the binary and run sender or receiver commands for your environment.

# Get the tool
git clone https://github.com/tsnperf/tsnperf-tool.git
cd tsnperf-tool

# Build the release binary
cargo build --release

# Start a receiver with the live dashboard
./target/release/tsnperf recv --web 0.0.0.0:8080 --output report.json
# Example multi-stream sender run
./target/release/tsnperf send \
  --streams "10:5,20:3,30:7" \
  --rate 1000 \
  --size 256 \
  --count 10000 \
  --cycle 1000us \
  --slot 250us

Consistent behavior across development systems and target hardware.

TSNPerf is positioned here as a dependable lab and field utility, whether you are validating from a Windows workstation or deploying to Linux ARM boards.

Windows

Keep the same feature set, reporting structure, and workflow across both platforms so test procedures remain straightforward and repeatable.
